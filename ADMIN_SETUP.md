## 🔧 Required Supabase Setup

### Step 1: Create Database Tables

Run these SQL commands in your Supabase dashboard:

```sql
-- User Profiles (tracks admin role)
CREATE TABLE user_profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  full_name VARCHAR,
  role VARCHAR DEFAULT 'user', -- 'admin' or 'user'
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Coupons Table (NEW - if not exists)
CREATE TABLE coupons (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  code VARCHAR NOT NULL UNIQUE,
  description TEXT,
  discount_type VARCHAR NOT NULL, -- 'percentage' or 'fixed'
  discount_value DECIMAL(10, 2) NOT NULL,
  minimum_order_value DECIMAL(10, 2),
  usage_limit INTEGER,
  usage_count INTEGER DEFAULT 0,
  is_active BOOLEAN DEFAULT true,
  expiry_date TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Knitwear Items Table (if not exists)
CREATE TABLE knitwear_items (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  title VARCHAR NOT NULL,
  description TEXT,
  image VARCHAR,
  collection VARCHAR,
  sort_order INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Enable RLS for security
ALTER TABLE user_profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE coupons ENABLE ROW LEVEL SECURITY;
ALTER TABLE knitwear_items ENABLE ROW LEVEL SECURITY;

-- Allow users to read own profile
CREATE POLICY "Users can read own profile" ON user_profiles
  FOR SELECT USING (auth.uid() = id);

-- Allow admins to manage coupons
CREATE POLICY "Admins can read coupons" ON coupons
  FOR SELECT USING (
    EXISTS (
      SELECT 1 FROM user_profiles 
      WHERE id = auth.uid() AND role = 'admin'
    )
  );

-- Allow public to read knitwear items
CREATE POLICY "Everyone can read knitwear" ON knitwear_items
  FOR SELECT USING (true);
```

### Step 2: Set Up Admin User

After creating the user_profiles table:

1. Sign up/login a user in your app
2. Go to your Supabase dashboard
3. Go to the `user_profiles` table
4. Insert a row with that user's ID and role='admin'

```sql
INSERT INTO user_profiles (id, full_name, role)
VALUES ('USER_ID_HERE', 'Mitali Dhumal', 'admin');
```

### Step 3: Set Up Storage Buckets

If not already created, create these in Supabase Storage:
- `products` - For product images and videos
- `blog` - For blog post images

## 🐛 Troubleshooting

**Problem: "Access Denied" on /admin**
- Ensure your user ID is in `user_profiles` table with role='admin'

**Problem: Can't upload images**
- Check Supabase storage bucket exists (products, blog)
- Check bucket is not restricted

**Problem: Admin tabs not loading**
- Check browser console for errors
- Ensure Supabase credentials are correct in .env.local
- Check internet connection

**Problem: Logout not working**
- This is now fully implemented - click Logout button on admin dashboard
